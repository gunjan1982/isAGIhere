# isAGIhere — Hard Rules for Claude Code

> These are non-negotiable. For project context, architecture, and current task state, read `docs/` instead.

## Session Start
Before any code work, read in this order:
1. `docs/PROJECT_CONTEXT.md`
2. `docs/TECH_CONTEXT.md`
3. Last block of the relevant `status.md` (`v1/status.md` or `v2/status.md`)

## Hard Rules

1. **Never use npm or yarn.** Always use `pnpm`. To add a dependency: `pnpm add [pkg] --filter @workspace/[artifact-name]`.

2. **Never edit `lib/api-client-react/` directly.** It is generated by Orval. To add a typed API hook: update `lib/api-spec/openapi.yaml` then run `pnpm --filter @workspace/api-spec run codegen`.

3. **Run DB migrations after every schema change.** Schema is in `lib/db/src/schema/`. After any change: `pnpm --filter @workspace/db run generate` → `pnpm --filter @workspace/db run migrate`. Verify a new file appears in `drizzle/migrations/`.

4. **`scripts/src/seed.ts` truncates tables.** Never run the seed script in production without explicit instruction. It is destructive.

5. **Admin routes check `gunjan1982@gmail.com`.** Do not change this without explicit instruction.

6. **Never fabricate data, outputs, or analysis.** If a command fails or data is missing, stop and report. Do not substitute placeholder values.

7. **Match the design language exactly.** All new UI uses: `font-mono`, SCREAMING_SNAKE_CASE labels, `bg-secondary`, `border-border`, `<Skeleton />` for loading states, `"NO_SIGNAL_FOUND"` empty states. Match existing components before creating new ones.

8. **Session end: append to status.md.** After any session that produces durable output, append a session block to the relevant workstream's `status.md`. See the template in `docs/PROJECT_CONTEXT.md` → Where To Go Next.

## What Does NOT Go Here
- Architecture, features, domain concepts → `docs/PROJECT_CONTEXT.md`
- Current bugs, build status, active tasks → `docs/TECH_CONTEXT.md`
- Session history and next actions → `v1/status.md` or `v2/status.md`
