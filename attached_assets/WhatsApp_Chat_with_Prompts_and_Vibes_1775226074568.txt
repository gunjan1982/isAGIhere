01/04/26, 3:20 pm - gunjan aggarwal: Here's a clear breakdown of **how these Anthropic leaks are actually happening** — based on the public details from the March 31, 2026 Claude Code incident and the earlier one a few days prior.

### 1. The Main Claude Code Source Code Leak (March 31, 2026)
This was **not a hack** or sophisticated breach. It was a straightforward **packaging and build configuration mistake** when publishing to npm (the JavaScript package registry).

- Anthropic released version **2.1.88** of `@anthropic-ai/claude-code` (their terminal-based AI coding agent).
- During the build process (using tools like Bun or similar), they generated a **source map file** (`cli.js.map`, ~59–60 MB).
- **Source maps** are normal developer tools: they map the final bundled/minified JavaScript back to the original readable TypeScript source code (including file paths, original variable names, comments, and even full `sourcesContent` with the entire code embedded as strings in JSON).
- The critical error: Their **.npmignore** file (which tells npm what *not* to include in the published package) did **not** exclude the `.map` files. So the huge source map shipped publicly with the package.
- Anyone who ran `npm install` or downloaded the tarball could extract the map and reconstruct ~1,900–1,906 original TypeScript files totaling over **512,000 lines** of clean, unobfuscated code.
- Some reports also mentioned the map pointing to (or being accompanied by) files in a public Cloudflare R2 bucket (e.g., paths like `r2.dev/src.zip`).

This is the **second time** it happened — a similar source map slip occurred in an earlier version (around February 2025). After the first incident, they removed the old version and the map, but the same oversight crept back in.

**Why it’s easy to miss**: Modern AI-assisted codebases are huge and heavily generated. Auditing every build artifact becomes harder, especially with defaults that enable source maps for easier debugging.

Anthropic quickly yanked version 2.1.88, issued takedowns on mirrors, and rolled back — but by then the code had already been mirrored on GitHub and spread widely.

### 2. The Recent "Mythos" (Capybara) Model Leak (Around March 26, 2026)
This was a completely different vector — **not related to code or npm**.

- Anthropic uses an external **Content Management System (CMS)** to prepare blog posts, drafts, assets (logos, papers, graphics), and internal docs for their website.
- Assets uploaded to their central data store were **public by default** unless explicitly marked private.
- Through a **configuration error** (described by Anthropic as "human error in the CMS configuration"), roughly **3,000 unpublished/draft files** became publicly accessible in their data lake or searchable storage.
- This included a draft blog post announcing their next big model (**Claude Mythos**, internally codenamed "Capybara"), performance claims, cybersecurity implications, and other internal notes.
- Security researchers and Fortune independently found and accessed it before Anthropic locked it down.

Again, no breach — just a misconfigured public storage/CMS setup.

### Why Are These Leaks Happening Repeatedly?
- **Operational carelessness at scale**: Even top AI labs move extremely fast (Anthropic had 14+ launches and multiple outages in March alone). Heavy use of their own AI coding tools (ironically, Claude Code itself) can accelerate development but also make codebases and build pipelines more complex and harder to manually audit.
- **Common pitfalls amplified**:
  - For npm: Forgetting to exclude debug artifacts (source maps, .ts files, etc.) in production builds is a classic JS ecosystem mistake.
  - For CMS/storage: Default-public settings + human config errors are frequent in cloud environments (Cloudflare R2, AWS S3-like buckets, etc.).
- **No evidence of intentional "poisoning"**: These are textbook accidental exposures. Anthropic is actively fixing them (removing versions, locking storage). Intentionally leaking your own IP would be self-defeating — it exposes architecture, prompts, internal flags (like KAIROS always-on mode, unreleased features), and engineering practices without any real legal barrier against competitors re-implementing ideas.

In short: These leaks stem from **human/config errors in release pipelines and content systems**, not malice or 4D chess. They reveal how even well-funded labs can trip over basic DevOps hygiene when shipping at breakneck speed with massive AI-generated code.

The silver lining for the community? It’s given developers a rare look inside a production agentic system (the Buddy pet, tool loops, safety prompts, etc.), even if Anthropic would rather it hadn’t happened.
