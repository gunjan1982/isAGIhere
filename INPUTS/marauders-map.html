THE HIDDEN CASTLE WINGS
8 Capability Wings

The matrix shows how you access AI. These wings are the underlying infrastructure, disciplines, and capabilities that power every Advanced and Expert cell.

🏰
Wing 1 — MCP Corridor
MODEL CONTEXT PROTOCOL

Open standard (Anthropic, now OpenAI + Google + Microsoft) that lets AI agents connect to any external tool via one universal plug. USB-C for AI integrations.

"Every tool used to build its own adapter. MCP is the one connector that works for everything."
MCP Host
The AI app (Claude, Cursor, Windsurf) that initiates connections
MCP Server
External tool wrapped in MCP: GitHub, Postgres, Slack, your own API
Claude Code
Supports MCP natively — connect to your DB without custom integration code
Cursor/Windsurf
Both have native MCP support for IDE-embedded tool connections
Matrix placement: 4-Advanced · 5-Advanced · 6-Expert
→ modelcontextprotocol.io

Wing 2 — Agent Frameworks Tower
CREWAI · AUTOGEN · LANGGRAPH · ADK

Frameworks for multi-agent systems — multiple AI agents with specialist roles working together on complex tasks. LangChain = one wizard. These = the Order of the Phoenix.

"Pick one: exploration → AutoGen. Production roles → CrewAI. Precise pipelines → LangGraph. Google Cloud → ADK."
LangGraph
Graph-based pipelines — precise state management, best for complex step-by-step flows
AutoGen

CrewAI

Google ADK

Matrix placement: 6-Advanced → 6-Expert
📚
Wing 3 — Vector DB Library
CHROMA · PINECONE · WEAVIATE · QDRANT

RAG needs somewhere to store and search document embeddings. Vector DBs store the meaning of text as mathematical coordinates — find things semantically similar, not keyword-identical.

"A regular DB stores rows. A vector DB stores meaning. Search by 'what this is about', not 'exact words'."
Chroma
Free, local, Python-native — start every RAG project here
Pinecone
Managed cloud, auto-scales, generous free tier — go-to for production
Weaviate
Hybrid semantic + keyword search, self-host or cloud
pgvector
Already on PostgreSQL? Add vector search with one extension
from langchain_community.vectorstores import Chroma vs = Chroma.from_documents(docs, OpenAIEmbeddings()) results = vs.similarity_search("anxiety management", k=5)
Matrix placement: 3-Intermediate · 6-Intermediate → Advanced
🔭
Wing 4 — LLMOps Observatory
HELICONE · LANGFUSE · LANGSMITH · W&B

Once your AI app is in production, you need to watch it. LLMOps tools give traces, cost tracking, eval pipelines, and debugging. Google Analytics — but for your AI.

"You wouldn't run a web app without analytics. Don't run an AI app without observability."
Helicone
One-line setup: change base_url and every call is logged + costed automatically
Langfuse
Open-source, self-hostable — full traces, eval pipelines, team dashboards
LangSmith
LangChain-native tracing + eval + prompt management
Weights & Biases
Experiment tracking for fine-tuning runs — the ML standard
# Helicone: 1 line = instant observability client = OpenAI(base_url="https://oai.helicone.ai/v1", default_headers={"Helicone-Auth": f"Bearer {key}"})
Matrix placement: 6-Intermediate → Expert
✍️
Wing 5 — Prompt Engineering Chamber
COT · FEW-SHOT · ROLE · TREE OF THOUGHT

Evolved from "write better questions" into a systematic discipline with named, repeatable techniques. 40–60% accuracy improvement with zero code changes or cost increases.

"The only wing that cuts across all 30 cells. Better prompts improve every mode, every level."
Chain of Thought
"Think step by step" — best for reasoning, maths, multi-step diagnosis
Few-shot
Show 2–3 examples before your question — 30–50% quality improvement
Role Prompting
"You are a clinical psychologist..." — activates domain-specific knowledge
Tree of Thought
AI explores multiple reasoning branches in parallel — complex problems
Matrix placement: ALL 6 columns, ALL 5 levels — this is the universal skill
🧬
Wing 6 — Fine-Tuning Laboratory
LoRA · QLoRA · PEFT · UNSLOTH

Re-train a model on your specific data to permanently specialise its behaviour — not just reference a textbook, but absorb the knowledge permanently.

"RAG = doctor with a textbook on the desk. Fine-tuning = doctor who went to medical school on your specific cases."
LoRA / QLoRA
Fine-tune small adapter layers only — runs on consumer GPU. The default method.
Unsloth
2–5× faster LoRA, free, open-source — the fastest way to fine-tune locally
HF PEFT
HuggingFace library — full menu of parameter-efficient fine-tuning methods
OpenAI FT API
Easiest start — GPT-3.5/4o-mini via API, no GPU, no infra needed
Matrix placement: 3-Expert · 6-Expert
